# Gentle Memories Progress

## Current system status

Initializer scaffold created. The project has a minimal TypeScript Obsidian plugin shell, a valid local-install Obsidian manifest, build tooling, an idempotent `init.sh`, and a smoke-test helper service.

Latest verification: `./init.sh` passed on 2026-04-30 after implementing F026. The default rendered preview for long memories is now shorter, preserving more of the note behind `Show more` while still showing enough opening content to orient the user. `Show more` still expands to the full rendered note body, and `Show less` returns to the shorter compact preview. The AI reading prompt control remains labeled `Memories` while preserving the same click behavior, privacy constraints, selected-provider API key validation, cache behavior, and user-triggered network request flow. AI prompt instructions still ask OpenAI and Claude to answer in the same primary language as the note excerpt and to produce a warm reading lead-in that may be a brief summary, reflection, encouragement, or gentle self-reflection question. Memory entries retain the source note body without frontmatter, and the memory modal renders that body through Obsidian Markdown rendering APIs instead of a plain excerpt paragraph. Generated or cached AI reading prompts render before the rich note content without any automatic AI request on startup or modal open. Debug mode defaults to disabled, is exposed in settings, reveals a settings-tab `Show memory` control only while enabled, and that control displays a memory immediately without startup scheduling or startup interval checks. Debug mode also emits privacy-safe developer console diagnostics for journal discovery counts, filter outcomes, selected memory path, and AI cache hits or misses without logging full note content, vault names, API keys, or request secrets. The repository includes `README.md`, `docs/manual-verification.md`, and `docs/orchestrator.md`. The manual verification checklist covers all ten scenarios from `SPEC.md` section 14 with setup, run, and expected-result steps for each scenario. `npm run verify:manual-plan` validates that the checklist stays aligned with the section 14 scenario list, and `init.sh` runs that validator after build and smoke checks. The registered `Gentle Memories: Show memory` command still builds displayable memories from eligible journal notes, derives a title, excerpt, content hash, and date when available, and shows an Obsidian modal with the title, optional date, rich rendered note content, and `Open note`, `Next`, `Close`, optional `Show more` or `Show less`, and, when AI is enabled, `Memories` buttons. Empty or unusable tagged notes are still skipped. Startup display still queues only when `showOnStartup` is enabled, waits for Obsidian layout readiness, records when a startup memory is shown, and skips future startup display until `minDaysBetweenStartupShows` has elapsed.
