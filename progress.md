# Gentle Memories Progress

## Current system status

Initializer scaffold created. The project has a minimal TypeScript Obsidian plugin shell, a valid local-install Obsidian manifest, build tooling, an idempotent `init.sh`, and a smoke-test helper service.

Latest verification: `./init.sh` passed on 2026-05-01 after implementing F041. The plugin now registers a user-facing `sparkles` ribbon icon with the sentence-case tooltip `Show memory`; clicking it calls the same manual memory display flow as the existing `Gentle Memories: Show memory` command, bypasses startup cooldown controls, and does not require debug mode. AI provider requests still use Obsidian's `requestUrl` API instead of global `fetch`, with OpenAI and Claude endpoints, headers, JSON request bodies, parsed `requestUrl` responses, non-2xx error handling, cache behavior, and privacy constraints preserved. AI response parsing and request construction no longer use unnecessary `async` method declarations when no `await` is needed. The settings tab title still uses `new Setting(containerEl).setName(...).setHeading()` instead of creating an HTML heading directly, but the heading no longer repeats the plugin name. Flagged Openai-facing user-visible settings text now uses scanner-compatible `Openai` casing in the provider dropdown, selected provider key label, missing-key notice, smoke coverage, and release-facing documentation while preserving the stored provider value `openai`, `openAiApiKey`, endpoints, headers, and selected-provider key behavior. Other flagged settings labels still use sentence-case wording, including `Enable AI lead-in`, `Cache AI lead-ins`, and the journal tags placeholder `Journal, diary, note`. The memory modal action row still wraps buttons cleanly on narrow screens, keeps button labels readable, and switches to full-width action rows on very narrow mobile screens so `Show more`, `Show less`, `Open note`, `Next`, `Close`, and `Memories` are not clipped. The repository also includes a GitHub Actions workflow that runs `npm ci` and `npm run build` on Ubuntu, Windows, and macOS for pushes to `main` and pull requests. Release-facing metadata remains consistent for the initial community plugin release. The default note preview remains shorter and constrained with a preview-height class so long notes stay close to a one-screen reveal until `Show more`; `Show less` restores the constrained preview. Automatic AI lead-in generation still shows `Loading memory lead-in...` immediately in the lead-in section, then replaces it asynchronously with the generated or cached text. Stale AI results from a previous memory are ignored after `Next`. Automatic and manual AI requests still send only the generated excerpt, not full note content, file paths, vault names, tags, display history, or cached history text. If AI is disabled, no AI button or network request appears. If AI is enabled but the selected provider key is missing, the modal does not make an automatic request or show repeated automatic missing-key notices; the `Memories` button remains available and shows the missing-key notice on manual click.
